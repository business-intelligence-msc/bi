Dimensions and Expressions

In order to create a chart in QlikView you need to understand what dimensions and expressions are.

When you make a chart, start by asking yourself "What do I want to show?" The answer is usually Sales, Quantity, or some other number. This is your expresssion, or more correctly, your measure. Next you should ask, how many times should this be calculated? Per what do I want to show this meausre? The answer could be per month, per customer, per supplier. This is your dimension.

An expression is simply a calculation that can be made over multiple records in your data set. The calculation returns one single value that summarises all relevant records. This is an aggreation, for example sum() count() avg() max() min().

Dimensions are descriptive attributes for example customer, product, location, supplier, activity, time, colour, size etc. The dimension is like the loop variable in a For loop, the calculation is done once per dimensional value. So the dimension determines how many rows, bars, points, or slices on a chart. The measure/expression is what is calculated in the loop.

Tips for Data Visualisation:

Prioritise the data.

Don't use effects without good reason (e.g. for emphasis). Usually you work within a limited screen area and every pixel of your application should be there for a reason. For example do you need a caption saying "Year" when you are presenting a list of 2012 2013 2014 etc?

Comparison.

One of the most effective visualisations you can choose for showing comparison is the bar chart. When comparing a set of items side by side to show quantitative behaviours use bar charts. When there are too many bars, labeling becomes problematic. It also implies the data is discrete, if your data continually changes over time use line graph instead. Avoid comparison of different units ($ and Qty) on the same scale. 3D bar charts are not intutive and lead the user to believe another dimension is involved.

Comparing different measurements. QlikView combo chart allows you to define a visualization style for each measure and you can use two axes with their own separate scale. (we did this in excel in a previous lab).

Tellis Charts

A trellis chart is a grid of small charts, a repetition of a chart across a grid, usually at least two dimensions are used, one for the grid variable (e.g. Sales area) and one for the x-axis within each small chart (e.g year). You can analyse the metrics within each chart and look at the bigger picture at the same time and identify any unusual behaviour in the variables. The y-axis is shared so you can compare the measures.

Sorting

Sorting is a special type of comparision. You can choose to define the primary sort order in the chart properties. Dimension limits can be used to limit the number of values shown useful for Top 5 or Bottom 5 performers.

Movements and Trends

In the majority of applications you need to anlayze your data over some sort of time dimension. Trends can be displayed effectively in a line chart. Ensure that a relation between the dimensions values exist and that they are in sequence and have the same interval between the data points. Eg sales and profits over months in a year.

Share and distribution

Pie charts are popular visualisations for displaying visual distribution of measurements and are used very frequently. Because the human mind has difficulty comparing areas (in this case the size of hte slices of hte pie) it is not recommended that you show a lot of data categories within the pie chart. It is also hard to apply colours that are far apart in hue from each other to clearly differentiate between the pie slices. Where the dimension values to be compared are less than seven a pie chart is suitable. It is also good practice to combine small slices into "others".

Correlation

Another key part of analysis is to discover if there is a correlation between distinct measures. The scatter chart is the perfect visualisation if you want to display correlation. Also a line chart with separate lines allows you to analyse correlation over time.

Order Trend

The CEO needs to see the trend of orders by month and wants to compare 2013 with 2012. Monitoring the order trend is key to the copany because the company experieinced a significant decrease of orders in 2013.

The number of orders measure is defined as count(distinct OrderId). Lets look at why this is the case. From the file menu choose table viewer, right click the facts table and then select preview. The facts table contains rows for every order line. Therefore the orderID is repeated for each order line. If we need to calculate the number of orders then we want to count distinct orderId's.

  • On the Dashboard sheet, create a new chart object. Name it Order Trend and do not show the title in the Chart, choose a line chart and click next.

Guage Chart

The guage chart has been in the BI dashboard industry for a long time and it is a very popular chart type. their general purpose is to determine the range of a given value. In a business dashboard it only works effectively in attracting the user's attention and doesn't work so well in getting hte point accross. A linear guage uses the space more effectively, and reducing the chart decoration makes the chart more intuitive.

The CEO needs to monitor the Margin% for the current year to date. Create a chart that displays the KPI Margin Percent based on the selection status. The chart must display the Margin Percentage, include segments for the target Margin Percentage, and clearly show which values fall above or below that point.

These are the Margin% KPI's target values:

Poor performance: lower bound = up to 10% Mediocre performance: Mid bound = up to 15% Good performance: Upper bound = over 15%

  • Create a new chart object on the dashboard sheet and choose gauge chart, click Next, click next again.
  • Create the expression Sum(Margin)/Sum(LineSalesAmount), click ok, for the label Margin%, click next and next again.
  • Select look and orientation as below:

  • Click next, on presentation clear autowidth segments, for Max type 0.5, under indicator, Mode select Fill to value.
  • Under show scale, set Major Units to 6. Clear Show Labels on Every, select Hide Segment Boundaries, select Hide Guage Outlines.
  • Define the segments:
    • Segment 1: Poor performance: lower bound = up to 10%
    • Segment 2: Mediocre performance: Mid bound = up to 15%
    • Segment 3: Good performance: Upper bound = over 15%
  • A segments width is defined by its lower bound and the lower bound of the next segment. Segment 1 lower bound = 0, segment 2 lower bound = 0.10, segment 3 lower bound = 0.15, min value = 0 max value = 0.5
  • For segment 1 select a lower bound colour, by clicking the colour tile and then under base colour click the colour tile and choose red. Click Ok twice.
  • set segment 2 lower bound colour to yellow. Create a new segment for segment 3 by clicking segments setup and Add, colour this a new greem colour.
  • Under reference lines select add, the reference lines dialog opens, for label type Goal. for Expression type 0.2 click ok.
  • Under text in chart, select Add, under text, type =Num(Sum(Margin)/Sum(LineSalesAMount),'##.##%'). click on Font, select Bold and size 11. click ok. click ok again.
  • Click next and next again.
  • In colours set transparency to 100%, click next.
  • In numbers set Margin% to Fixed to 2 decimal places and display as percent.
  • Click next until you come to the caption tab, clear show caption, click finish.

  • Press Ctrl+Shift and use your mouse to resize and move the guage for best diplay and fit.

Create a linear guage with dynamic border bounds. In this guage, two segments are sufficient, one for poor (below 15%) and the other for good performance.
